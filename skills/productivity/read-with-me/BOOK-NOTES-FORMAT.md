# BOOK-NOTES-FORMAT.md

单本书的累积笔记,章节追加。**三区结构**(Zettelkasten)在单文件内的分区实现。`books/<书名>/NOTES.md` 使用此格式。

## 三区结构

### 区 1:闪念区(Fleeting Notes)

读时随手记的念头/疑问/困惑。粗糙、临时,后续可加工或丢弃。

格式:按日期分块,每条闪念一行。

```markdown
## 区 1:闪念

### 2026-06-13(第 N 章)
- 这段让我想到 X,但还没想清楚关联
- 作者这个类比我不懂,需要查 Y
- 疑问:为什么 Z?
```

**维护**:每次陪读收尾时,扫一遍本日的闪念——值得深挖的加工进文献/永久区,不值得的保留作为"当时的困惑"记录(可能未来会解开)。

### 区 2:文献笔记区(Literature Notes)

从划线/原文提炼。**强制规则**:

1. **用自己的话**浓缩,禁止照抄划线原文(weread 的 `markText` 字段)
2. **记出处**:章节 + 位置(range) + 时间
3. 每条文献笔记**一个概念/命题**

格式:

```markdown
## 区 2:文献笔记

### L001 — [概念/命题名]
- **出处**:第 N 章,range 1234-2345(2026-06-13 划的)
- **原文要点**:[一句话,用自己的话,不抄原文]
- **为什么重要**:[一句话,你当时划它的动机]
- **相关**:L002,三体-P003
```

### 区 3:永久笔记区(Permanent Notes)

用「三问」加工成的核心想法。每条**给唯一编号**(如 `书名首字母缩写-P001` 或直接 `书名-P001`),可被其他书的笔记跨书引用。

格式:

```markdown
## 区 3:永久笔记

### 三体-P001 — [你的命题,不是作者的]
- **来源文献**:L001, L005
- **加工三问**:
  1. 反驳/支持/补充已知?[答]
  2. 能否组合出新东西?[答]
  3. 触发了什么新问题?[答]
- **正文**:[2-5 句,完整段落,像在写给未来的自己看]
- **contradicts**(如有):[已知命题编号,如 三体-P002 或其他书 某书-P001]
- **related**:三体-P003, 某书-P007
- **日期**:YYYY-MM-DD
```

## 追加规则

- 每次陪读收尾,skill 评估本次对话产出,决定写入哪区:
  - 困惑/未解问题 → 闪念区
  - 对作者某段话的翻译(用自己的话)→ 文献笔记区
  - 经过三问加工的核心想法 → 永久笔记区
- 永久笔记的编号**全书唯一、永不复用**(即使删除某条,编号保留)
- 跨书引用直接写 `其他书-PXXX` 编号,链接到 `../其他书/NOTES.md#其他书-PXXX`

## 编号规则建议

- 文献笔记:`L001, L002, ...`(全书连续)
- 永久笔记:`<书名>-P001, <书名>-P002, ...`(书名取简称)
- 闪念:无编号,按日期分块
