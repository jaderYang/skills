# PROFILE-FORMAT.md

阅读偏好/目标/口味档案。根层 `~/reading/PROFILE.md` 使用此格式。

## 字段说明

| 字段 | 必填 | 说明 |
|------|-----|------|
| `口味` | ✓ | 虚构 vs 非虚构偏好,哪些领域(如历史、金融、科技、文学) |
| `目标` | ✓ | 读书的动机/目的(如职业成长、兴趣、拓宽视野、写作输入) |
| `每周时间` | ✓ | 预估每周阅读小时数,用于推算节奏 |
| `陪读强度` | ✓ | `温和` / `中等` / `grill`,决定追问的硬度 |
| `已掌握主题` | ✗ | 用户自报已经熟悉的领域,影响串联难度 |
| `想突破主题` | ✗ | 用户想啃硬骨头的方向 |
| `禁忌/不想读` | ✗ | 明确拒绝的内容类型 |

## 模板

```markdown
# 阅读档案

## 口味
- 偏好:[虚构 / 非虚构 / 混合]
- 领域:[领域1, 领域2, ...]

## 目标
[用一两段话描述读书的动机与期待]

## 每周时间
约 [X] 小时

## 陪读强度
[温和 / 中等 / grill]

## 已掌握主题(用于串联难度)
- ...

## 想突破主题
- ...

## 禁忌/不想读
- ...

## 口味变化日志
- [YYYY-MM-DD] 变化描述
```

## 维护规则

- 首次 onboarding 时由 skill 根据用户回答填写
- 用户在后续陪读中表达新偏好时,追加到「口味变化日志」
- 不删除历史,只追加
