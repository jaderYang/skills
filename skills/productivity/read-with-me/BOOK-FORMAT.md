# BOOK-FORMAT.md

单本书的元信息档案。`books/<书名>/BOOK.md` 使用此格式。

## 必填字段

| 字段 | 说明 |
|------|-----|
| `bookId` | weread bookId(若书不在微信读书,留空) |
| `书名` | 书名 |
| `作者` | 作者 |
| `书类型` | `非虚构` 或 `虚构` — 决定陪读招式切换 |
| `为什么读` | 用户的 mission(对标 teach 的 MISSION) |
| `想得到什么` | 用户期望从这本书获得什么 |
| `开始日期` | YYYY-MM-DD |
| `当前状态` | 在读 / 读完 / 暂停 |

## 选填字段(检视阶段填充)

| 字段 | 说明 |
|------|-----|
| `全书地图` | 检视阅读后画出的全书结构:部分→章→核心命题 |
| `核心问题` | 这本书要解决的核心问题是什么 |
| `作者背景` | 一句话作者定位(已查证) |
| `相关书` | 与本书相关的其他书(主题阅读的素材),链接到 `books/<他书>/BOOK.md` |
| `上次进度` | 上次陪读结束时的进度(章节/百分比),每次收尾更新 |
| `平均阅读量` | 根据历史会话算出(章节数/百分比),用于推算下次建议 |

## 模板

```markdown
# BOOK:《书名》

## 元信息
- **bookId**:[weread bookId,可为空]
- **书名**:书名
- **作者**:作者
- **书类型**:[非虚构 / 虚构]
- **开始日期**:YYYY-MM-DD
- **当前状态**:[在读 / 读完 / 暂停]
- *上次进度*:[章节/百分比,每次收尾更新]
- **平均阅读量**:[章节数/百分比,从第4次陪读起开始计算]

## Mission:为什么读这本书
[用户的 mission,对标 teach 的 MISSION.md]

## 想得到什么
[用户期望获得什么——知识/技能/视野/情感共鸣]

## 全书地图(检视阶段产出)
```
Part I: ...
  Ch 1: ... — 核心命题:...
  Ch 2: ... — 核心命题:...
Part II: ...
  ...
```

## 核心问题
[这本书要解决的核心问题]

## 作者背景
[一句话作者定位,已查证]

## 相关书
- [《他书1》](../他书1/BOOK.md) — 关联点:...
- [《他书2》](../他书2/BOOK.md) — 关联点:...
```

## 维护规则

- 首次陪读某书时由 skill 根据 weread 数据 + 用户回答 + 检视阅读创建
- 用户在后续陪读中修正 mission 或状态变化时,更新对应字段并记录「状态变化日志」
- 状态变化日志格式:`[YYYY-MM-DD] 字段名:旧值 → 新值(原因)`
