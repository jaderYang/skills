# 方法论引擎

三套读书/学习方法论，转化为陪读的可执行机制。所有方法论源登记在 `RESOURCES.md`，引用时给出出处。

## Adler 四层阅读法 → 三种陪读模式

- **检视模式(Inspectional)**: 开新书时先做。读目录/序/结构，回答"这本书值不值得深读" + 画出全书地图。落地在 BOOK.md 的"全书地图"字段。
- **分析模式(Analytical)**: 深读的主循环。套用**「苛求读者四问」**作为追问骨架:
  1. 这一章/这段在谈什么？(主旨)
  2. 细节上说了什么、怎么论证的？(结构)
  3. 说得对吗？全对还是部分对？(批判)
  4. 跟你有什么关系？(so what / 对你的 mission 意味着什么)
- **主题模式(Syntopical)**: 跨书串联——把当前书的概念和读过的其他书对照、translate 成自己的术语、提炼自己的问题。落地在 `GLOSSARY.md` 和永久笔记的跨书引用。

## Zettelkasten → 笔记三区

`NOTES.md` 内分三区（详见 BOOK-NOTES-FORMAT.md）:

- **闪念区(Fleeting)**: 读时随手记的念头/疑问，粗糙、临时。
- **文献笔记区(Literature)**: 从划线提炼——**强制用自己的话浓缩 + 记出处**，禁止照抄原文。
- **永久笔记区(Permanent)**: 用「三问」加工:
  - 新信息如何反驳/支持/补充已知？
  - 能否组合出新东西？
  - 触发了什么新问题？

每条永久笔记给唯一编号（如 `三体-P003`），可跨书引用。**刻意保存矛盾观点**（标注 `contradicts: <已知>`），对抗确认偏误。

## 循证学习法 → 复述与复习

- **费曼式复述**: 让用户用大白话讲给"外行"听；讲不清 = 理解断点。
- **主动回忆 + 间隔重复**: HTML 卡片在 `READING-LOG.md` 登记复习时间表（间隔递增: 1天/3天/7天/30天），每次开场挑到期卡片抽考。

## 讨论招式映射

| 招式 | 方法论来源 | 何时触发 |
|------|-----------|---------|
| 复述确认 | 费曼 + 主动回忆 | 用户带划线回来讨论时 |
| grill 式追问 | Adler 四问 | 围绕用户有感触的点展开 |
| 串联前文/跨书 | Adler 主题阅读 + Zettelkasten | 出现与前文/他书相关概念时 |
| 思辨碰撞 | 矛盾观点 + 热门划线 | 作者下判断/用户提出观点时 |
| 推荐阅读 | weread `/store/search` | 用户明显无知或错误时 |
| 笔记沉淀 | Zettelkasten 三区 | 每次收尾 |
| 复习抽考 | 间隔重复 + 测试效应 | 每次开场 |
| 背景补充 | 不信任脑内知识 | 用户问"什么意思/背景"时 |
| 阅读规划 | 进度追踪 + 节奏推算 | 每次开场 + 收尾 |

**重要**: 所有背景补充/事实陈述**必须给出出处**（weread 接口名 / RESOURCES / 外部 URL）。不信任脑内知识。
